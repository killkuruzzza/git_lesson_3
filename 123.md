## Сохранения

**git add** - индексирование

**git commit** - сохраняет состояние репозитория

**git commit -m “мой комментарий”** - сохраняет состояние репозитория с моим комментарием

**git rebase**  ****- выстраивает ровную последовательность коммитов

## Ветки

**git log** - показывает коммиты в выбранной ветке

**git log - -oneline** показывает коммиты в выбранной ветке в сокращенном виде

**git log - -oneline - - graph** показывает коммиты в выбранной ветке в сокращенном виде+ графику комитов

**git branch [yourbranchname]**  - создание новой ветки

**git checkout [yourbranchname]** переход на указанную ветку

**git checkout -b [yourbranchname]** - переключение на новую ветку

**git branch -d [yourbranchname]** - удаление выбранной ветки

**git merge** **[yourbranchname]** - слияние веток, создает новый объединенный коммит.

**git checkout HEAD^** - возвращает нас к предыдущему коммиту на выбранной ветке

**git checkout [хеш_номер коммита]** - возвращает нас к нужному коммиту, к нужной 

**git branch -f [main] [HEAD~3] -** Переместит (принудительно) [выбранную ветку `main`]
 на три родителя назад от `HEAD`(колич-во перемещений задается после знака ) Вместо **[HEAD~3] можно использовать [HEAD^] или [номер коммита]**

**git reset HEAD^** - удаляет последний коммит на выбранной ветке(работает локально)

**git revert** **[yourbranchname]** - поделиться отменёнными локальными изменениями с удаленным сервером,

• `git cherry-pick <Commit1> <Commit2> <...>` - позволяет собрать в 1 финальный коммит несколько коммитов из различных веток

**git rebase -i** **HEAD~4 -**  работает также как **git chery-pick ,** можно выбрать что оставить или удалить из 4 последних коммитов, переставлять коммиты.

**git commit --amend -** внести изменения в прошлый коммит после его переставления

### Удаленный репозиторий

**git clone [ссылка github]** - создает локальную копию удаленного репозитория

**git fetch** - связывается с указанным удалённым репозиторием и забирает все те данные проекта, которых у вас ещё нет. Должны появиться ссылки на все ветки из этого удалённого репозитория (например, `origin/main`) 

*Важно отметить, что команда `git fetch`забирает данные в ваш локальный репозиторий, но не сливает их с какими-либо вашими наработками и не модифицирует то, над чем вы работаете в данный момент. потребуется вручную слить эти данные с вашими, когда вы будете готовы.*

Что бы вручную соединить ветки мы можем выполнить одну из команд:

- `git cherry-pick origin/main`
- `git rebase origin/main`
- `git merge origin/main`

**git pull** - команда скачивания изменений с удаленного репозитория и  объединения локальным репозиторием

**git fakeTeamwork [2]-**  [2] фальшивых коммита на удаленной ветке

**git push** - отвечает за загрузку локальных **изменений (наших коммитов)в указанный удалённый репозиторий, можно рассматривать как "публикацию" своей работы.
