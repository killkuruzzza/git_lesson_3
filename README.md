## I. **Разде 1 - Подготовка**
### **1. Подготовительный этап.**
### 1.1 *Установка ПО*
 Для начала работы необходимо установить две программы:
* GIT
* Visual studio Code
 
В данном руководстве мы не будем детально описывать методику установки, т.к. это процес интуитивно понятен и не вызывает каки-либо трудностей. Основным условием является установка программ с официальных сайтов разработчиков.
 
### 1.2. *Первый запуск*
В удобном месте необходимо создать папку в которой будет хранится наш проект.
 
После этого необходимо открыть программу Visual studio cod и выбрать пункт меню **File --> Open Folder**. После этого папку появится в области программы **Explorer**. После этого в этой области необходимо навести курсор на название папки и выбрать иконку создания файла **New file**.
 
> Важно помнить, что при создании файла его название должно заканчиваться следующими символами **.md**
 
После создания файла откроется область редактирования. И при нажатии кнопок **CTRL + Shift + V** откроется область предварительного просмотра, в которой будет отображаться информация в финальном виде (как это будет видеть пользователь).
 
Последним, что необходимо сделать на данном этапе, является открытие терминала, для этого необходимо выбрать соответствующий пункт меню **Terminal--> New Terminal**, после этого внизу окна откроется область для внесения команд.

## II. **Раздел 2 - Основные функции**
### 1. Использование команды git init.
Данная команда используется один раз при создании папки, чтобы инициализировать папку в которой в последующем будут сохраняться все изменения.Иными словами, после того как мы создали папку и первый файл необходимо создать репозиторий при помощи данной команды.
 
### 2. Использование команды git status.
При вводе данной команды отображается информация о файлах в репозитории. Если вы только создали файл и ввели команду **git status** вы увидите, что красным цветом выделен созданный вами файл и он является еще неотслеживаемым. Для того, чтобы Git начал его отслеживать необходимо ввести команду **git add + название файл**. После использования данной команды при повторном вводе **git status** название файла будет отображаться зеленым цветом.
> Для удобства ввода функции можно использовать стрелки на клавиатуре вверх и вниз, для прокрутки предыдущих команд, т.к. они все сохраняются в памяти.
 
### 3. Использование команды git add.
Данная команда используется для того, чтобы включить созданный файл в область подготовленных файлов. Иными словами, после этого GIT начнет отслеживать все внесенные изменения в данный файл. Использовать эту функцию необходим каждый раз после сохранения
>Важно помнить, что перед использованием функции **git add** необходимо сохранить все сделанные изменения в файле, для этого проще всего воспользоваться командой **CTRL + S**.
 
> наиболее удобный формат использования команды **git add** является использование **tab** при заполнении имени файла, его использование упрощает процесс за счет автозаполнения названия по первым буквам.
Есть также еще несколько способов использования данной команды, но в данном руководстве они не будут рассматриваться.
 
### 4. Использование команды git commit.
Это следующая часто используемая команда, которая позволяет, так сказать делать сохранения или иными словами коммит. Проще говоря при вводе данной команды делается запись об внесенных изменения, в последствии при использовании команд, которые мы рассмотрим далее, возможно будет перемещение между версиями файла.
Так же дя удобства последующей навигацией между версиями файла удобно оставлять тестовую информацию о внесенных изменения.
Таким образом команда используется после команды **git add** и вводится следующим образом **git commit -m"Сообщение описывающее внесенные изменения"**
 
### 5. Использование команды git log.
Команда используется для того чтобы отслеживать различные версии файла. Вызывается данная информация путем ввода команды *git log*. По результатам ввода, мы можем посмотреть все версии файла, увидеть комментарии, которые были оставлены к этому файлу и перейти к любой из версий файла, для этого достаточно ввести **git checkout + первые 4 цифры кода коммита**. Чтобы вернуться к финальной версии достаточно ввести **git checkout master**
 
### 6. Использование команды git branch.
Данная команда используется для создания и перемещением между несколькими ветками файла.  Так для создания второй ветки достаточно ввести команду **git branch second**. В последствии можно переключаться между этими ветками по аналогии с использованием команды **git log**. Более детальное описание данной функции мы с вами рассмотрим в обновленной версии руководства.

## III. **Раздел 3 (Формат текста в Markdown)**
 
1. **Выделение шрифта**
 
Заголовок первого уровня выделяется одинарной решеткой **#** с последующим пробелом. Заголовок второго уровня выделяется аналогичным способом, единственное отличие, что перед ним ставится 2е решетки **##** Подобным образом можно выделять заголовки и 3его и 6 го уровня
 
2. **Списки**
 
Списки бывают нумерованные и ненумерованные. Чтобы сделать нумерованный список достаточно поставить цифру перед текстом и точку, например вот так "1. ......". Если нужно сделать не нумерованные списки достаточно поставить * и пробле вот так "* ......".
 
3. **Работа с изображением**
 
     Для того, чтобы вставить изображение достаточно для начала поместить изображение в папку, где создан репозиторий и после этого ввести "![......](......). В квадратных скобках указывает текст, который будет отображаться в случае отсутствия изображения. В круглых скобках указываем название файла, вместе с расширение.  
 
4. **Ссылки**
 
    Ссылки вставляются обычным способом, как и во всех программах и офисных приложениях. Просто копируется ссылка и вставляется в текст, например вот так https://habr.com/ru/post/541258/
 
5. **Таблицы**
Таблицы строятся при помощи "|" - нужны для создания столбцов и "---" для создания первой строки. Пример можно посмотреть ниже.
 
|№| НАзвание | Цена|
|---|---|---|
|1| Морковь|5 руб|
|2| Помидор|10 руб|
|3|Огурец|15 руб|
 
 
    6. **Цитаты**
 
Чтобы сделать цитату, достаточно просто перед ней поставить вот этот значок ">" в начале строки и начать печатать текст

## IV. **Раздел 4 (Удаленный репозиторий)**
 
 1. **git clone**
 
Команда "git clone + адрес из github" - чтобы сделать не локальный репозиторий, а скачать чей-то репозиторий.
> После этого нужно ввести в терминале команду "cd + название скачанной директории", т.к. если этого не сделать мы останемся в той папке которую создали в самом начале и не сможем работать с клоном.
 
2. **git remote**
 
Команда "git remote + адрес" - связь локального и удаленного репозитория.
 
3. **git push**
 
Команда "git push -u" - даем команду протолкнуть, что мы сделали локально и отправить в удалённый репозиторий.
 
 
4. **git pull**
 
Команда "git pull" - стянуть с удаленного репозитория (из интернета) более новую версию для локальной. Команда составная, поэтому он делает merge.
 
> Если мы хотим поучаствовать в проекте (чужом) заходим на сайте github в нужный репозиторий и делаем его точную копию, при помощи кнопки fork. Всегда предложения для чужого проекта делаются в отдельной ветке
